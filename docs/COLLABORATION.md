# 複数端末での協働

## 推奨: オンライン提出（Cloudflare）

写真は **Cloudflare R2** に保存します。手順は [EVENT_SERVER_ARCHITECTURE.md](./EVENT_SERVER_ARCHITECTURE.md) を参照してください。

1. 代表者: イベント作成（オンライン提出オン）→ **招待 URL** を配布  
2. 参加者: URL から写真付き衣装を提出  
3. 代表者: 「サーバーから取り込む」→ 最適化  

## 予備: JSON ファイル（オフライン）

API 未設定時や障害時用。衣装コーディネーターは端末内 **IndexedDB** にも保存します。  
QR だけでは写真は届きません。

## フロー

### 1. 代表者：イベント作成

1. 「イベントを作成する」でイベントを登録
2. イベント詳細 → **① 参加用ファイルを書き出す**（`event-invite-{id}.json`）
3. LINE 等で参加者に送る（QR は補助。ファイル必須）

### 2. 参加者：参加・衣装登録

1. トップ → **イベントに参加（ファイル）**（`/join`）
2. 参加用 JSON を読み込む
3. 「衣装を追加する」で自分の衣装・写真を登録
4. イベント詳細（または代表者が案内した手順）で **③ 提出用ファイル**（`participant-submission-…json`）を書き出し、代表者へ送る

### 3. 代表者：提出の取り込み・最適化

1. イベント詳細 → **② 参加者の提出ファイルを取り込む**
2. 衣装・参加者名・希望順がマージされる
3. 「最適化」で割り当て → 結果を共有

## 同一端末だけで使う場合

代表者が全員分の衣装を登録し、参加者名を手入力しても問題ありません。

## クラウド同期（Google / Dropbox）

- **同じアカウント** の端末間でのバックアップ・復元用です
- 参加者ごとのリアルタイム共有や、別アカウント間の協働には使えません
- 設定画面から接続
