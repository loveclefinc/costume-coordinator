# Costume Coordinator - モバイルアプリデザイン仕様

## デザイン原則

本アプリは**iOS Human Interface Guidelines (HIG)準拠**の、洗練されたファーストパーティアプリのような体験を提供します。

- **モバイルポートレート前提** (9:16)
- **片手操作を重視** (主要アクションは画面下部1/3に配置)
- **視覚的な明快さ** (衣装画像を大きく、情報は簡潔に)
- **直感的なフロー** (撮影→解析→登録が3タップ以内)

## 画面構成

### 1. ホーム画面 (Home)

**目的**: 登録済み衣装の一覧表示と素早いアクセス

**レイアウト**:
- **ヘッダー**: 
  - タイトル「My Costumes」(Large Title)
  - 右上: 設定アイコン
- **メインコンテンツ**:
  - 衣装カードグリッド (2列)
  - 各カード: サムネイル画像 + 衣装名 + 主要色インジケーター
  - 空状態: 「衣装を追加しましょう」+ 大きな追加ボタン
- **フローティングアクションボタン (FAB)**:
  - 右下固定: 「+」アイコン (衣装追加)
  - サイズ: 56pt × 56pt
  - 色: アクセントカラー

**ユーザーフロー**:
1. 衣装カードをタップ → 衣装詳細画面へ
2. FABをタップ → 衣装追加画面へ

---

### 2. 衣装追加画面 (Add Costume)

**目的**: 写真撮影または選択、自動解析、メタデータ入力

**レイアウト**:
- **ヘッダー**: 
  - 左上: 「キャンセル」ボタン
  - タイトル「新しい衣装」
  - 右上: 「保存」ボタン (入力完了後に有効化)
- **メインコンテンツ** (スクロール可能):
  1. **画像選択エリア** (画面幅いっぱい、4:3比率)
     - 未選択時: 破線枠 + カメラアイコン + 「写真を撮る / 選ぶ」
     - 選択後: 画像表示 + 右上に「変更」ボタン
  2. **自動解析結果セクション**
     - 見出し「解析結果」(Title)
     - 主要色: カラーチップ (48pt × 48pt) + HEX値
     - 色系統: チップ表示 (暖色/寒色/中間)
     - トーン: チップ表示 (パステル/ビビッド等)
     - 各項目の右に「編集」アイコン (タップで手動修正)
  3. **衣装名入力**
     - テキストフィールド (プレースホルダー: 「例: 赤いドレス」)
  4. **柄選択**
     - セグメントコントロール: 無地 / 花柄 / ストライプ / ドット / その他
  5. **タグ入力** (任意)
     - チップ入力フィールド (例: レース、ロング丈)
- **ボトムボタン**:
  - 「保存」ボタン (画面幅いっぱい、Safe Area考慮)

**ユーザーフロー**:
1. 画像選択エリアをタップ → カメラ/ギャラリー選択
2. 画像選択後、自動解析が実行される (ローディング表示)
3. 解析結果を確認・修正
4. 衣装名・柄・タグを入力
5. 「保存」をタップ → ホーム画面に戻る

---

### 3. 衣装詳細画面 (Costume Detail)

**目的**: 衣装の詳細情報表示、編集、使用履歴確認

**レイアウト**:
- **ヘッダー**:
  - 左上: 「戻る」ボタン
  - 右上: 「編集」ボタン
- **メインコンテンツ** (スクロール可能):
  1. **画像表示** (画面幅いっぱい、4:3比率)
  2. **衣装名** (Title)
  3. **メタデータカード**
     - 主要色 / 色系統 / トーン / 柄
     - 各項目をアイコン + ラベルで表示
  4. **タグ** (チップ表示)
  5. **使用履歴セクション**
     - 見出し「使用履歴」
     - リスト: イベント名 + 日付
     - 空状態: 「まだ使用していません」
- **ボトムボタン**:
  - 「削除」ボタン (赤色、破壊的アクション)

**ユーザーフロー**:
1. 「編集」をタップ → 編集モード (衣装追加画面と同様のUI)
2. 「削除」をタップ → 確認ダイアログ → 削除実行 → ホーム画面に戻る

---

### 4. イベント一覧画面 (Events)

**目的**: 参加中・過去のイベント一覧表示

**レイアウト**:
- **ヘッダー**:
  - タイトル「イベント」(Large Title)
  - 右上: 「+」ボタン (イベント作成)
- **メインコンテンツ**:
  - イベントカードリスト (縦スクロール)
  - 各カード: イベント名 + 日付 + 参加人数 + ステータスバッジ
  - 空状態: 「イベントを作成またはURLから参加しましょう」
- **ボトムボタン**:
  - 「招待URLから参加」ボタン

**ユーザーフロー**:
1. イベントカードをタップ → イベント詳細画面へ
2. 「+」をタップ → イベント作成画面へ
3. 「招待URLから参加」をタップ → URL入力ダイアログ → 参加処理

---

### 5. イベント作成画面 (Create Event)

**目的**: イベント情報と条件を設定

**レイアウト**:
- **ヘッダー**:
  - 左上: 「キャンセル」ボタン
  - タイトル「新しいイベント」
  - 右上: 「作成」ボタン
- **メインコンテンツ** (スクロール可能):
  1. **基本情報**
     - イベント名入力
     - 日付選択 (DatePicker)
  2. **条件設定セクション**
     - 見出し「衣装の条件」
     - 色系統: セグメントコントロール (指定なし/暖色/寒色/中間)
     - トーン: セグメントコントロール (指定なし/パステル/ビビッド/ダーク)
     - 花柄ルール: スイッチ「花柄を許可」+ 数値入力「上限人数」
     - 同系色回避: スイッチ
     - 直近使用除外: 数値入力「◯日以内」(デフォルト30)
- **ボトムボタン**:
  - 「作成」ボタン

**ユーザーフロー**:
1. イベント情報と条件を入力
2. 「作成」をタップ → イベント詳細画面へ遷移 + 招待URL生成

---

### 6. イベント詳細画面 (Event Detail)

**目的**: イベント情報表示、衣装選択、最適化実行

**レイアウト**:
- **ヘッダー**:
  - 左上: 「戻る」ボタン
  - タイトル: イベント名
  - 右上: 「共有」ボタン (招待URL共有)
- **メインコンテンツ** (スクロール可能):
  1. **イベント情報カード**
     - 日付 / 参加人数 / ステータス
  2. **自分の衣装選択セクション**
     - 見出し「あなたの候補衣装」
     - 第1〜第5希望の衣装カード (タップで選択)
     - 各カード: サムネイル + 衣装名 + 希望順位バッジ
     - 「衣装を追加」ボタン (ホーム画面の衣装一覧から選択)
  3. **参加者セクション**
     - 見出し「参加者」
     - 参加者リスト (名前 + 衣装提出状況)
  4. **最適化結果セクション** (実行後のみ表示)
     - 見出し「提案された組み合わせ」
     - 提案カードリスト (スコア順)
     - 各カード: 参加者名 + 衣装サムネイル + 希望順位
- **ボトムボタン**:
  - 「最適化を実行」ボタン (全員が衣装提出後に有効化)

**ユーザーフロー**:
1. 「衣装を追加」をタップ → 衣装選択モーダル → 希望順位設定 → 追加
2. 「共有」をタップ → 招待URL共有シート (SNS/メール等)
3. 「最適化を実行」をタップ → ローディング → 最適化結果表示
4. 提案カードをタップ → 提案詳細モーダル (全員の割り当て確認)

---

### 7. 設定画面 (Settings)

**目的**: アプリ設定、データ管理

**レイアウト**:
- **ヘッダー**:
  - 左上: 「戻る」ボタン
  - タイトル「設定」
- **メインコンテンツ** (リスト形式):
  1. **表示設定**
     - ダークモード切り替え
  2. **データ管理**
     - 「全衣装データをエクスポート」
     - 「全衣装データをインポート」
     - 「全データを削除」(赤色、確認ダイアログ)
  3. **アプリ情報**
     - バージョン情報
     - 利用規約
     - プライバシーポリシー

---

## 主要ユーザーフロー

### フロー1: 初回衣装登録
1. ホーム画面 (空状態)
2. FABをタップ
3. 衣装追加画面
4. 写真撮影/選択
5. 自動解析 (3秒以内)
6. 結果確認・修正
7. 保存
8. ホーム画面に戻る (衣装カード表示)

### フロー2: イベント作成→最適化
1. イベント一覧画面
2. 「+」をタップ
3. イベント作成画面
4. イベント情報・条件入力
5. 作成
6. イベント詳細画面
7. 「共有」で招待URL送信
8. 自分の衣装選択 (第1〜第5希望)
9. 他参加者も衣装選択
10. 「最適化を実行」をタップ
11. 最適化結果表示 (複数案)
12. 提案を選択・確定

### フロー3: 招待URLから参加
1. 招待URLを受信 (メール/SNS等)
2. URLをタップ → アプリ起動
3. イベント詳細画面に遷移
4. 自分の衣装選択
5. 提出完了

## インタラクション

### アニメーション
- **画面遷移**: スライドイン/アウト (300ms)
- **カードタップ**: スケール 0.95 + 透明度 0.9 (150ms)
- **FAB**: タップ時にバウンス効果
- **ローディング**: スピナー + 「解析中...」テキスト

### ハプティックフィードバック
- **ボタンタップ**: Light Impact
- **衣装追加成功**: Success Notification
- **エラー発生**: Error Notification
- **最適化完了**: Success Notification

### ジェスチャー
- **衣装カード**: 長押しで削除メニュー表示
- **画像表示**: ピンチズームで拡大
- **モーダル**: 下スワイプで閉じる

---

## コンポーネント設計

### 1. CostumeCard
- **サイズ**: (画面幅 - 48pt) / 2
- **内容**: サムネイル + 衣装名 + 主要色インジケーター (下部に8pt高さのカラーバー)
- **状態**: デフォルト / 選択中 / 無効

### 2. EventCard
- **サイズ**: 画面幅 - 32pt
- **内容**: イベント名 + 日付 + 参加人数アイコン + ステータスバッジ
- **状態**: 進行中 (緑) / 終了 (グレー)

### 3. ColorChip
- **サイズ**: 48pt × 48pt (詳細表示) / 24pt × 24pt (リスト表示)
- **形状**: 円形
- **ボーダー**: 1pt グレー

### 4. PriorityBadge
- **サイズ**: 24pt × 24pt
- **形状**: 円形
- **色**: 第1希望=金、第2=銀、第3=銅、第4-5=グレー
- **内容**: 数字 (1-5)

### 5. FloatingActionButton (FAB)
- **サイズ**: 56pt × 56pt
- **位置**: 右下から16pt、下部Safe Areaから16pt
- **色**: アクセントカラー
- **シャドウ**: elevation 6

## レスポンシブ対応

- **小型端末 (iPhone SE等)**: カードサイズを調整、余白を縮小
- **大型端末 (iPad等)**: 3列グリッド、サイドバーナビゲーション
- **ランドスケープ**: 非推奨 (ポートレート固定を推奨)

---

## アクセシビリティ

- **VoiceOver対応**: 全要素にアクセシビリティラベル設定
- **ダイナミックタイプ**: システムフォントサイズに追従
- **コントラスト比**: WCAG AA準拠 (4.5:1以上)
- **タッチターゲット**: 最小44pt × 44pt

---

## エラー状態

- **ネットワークエラー**: トースト通知 + リトライボタン
- **画像解析失敗**: 「手動で入力してください」メッセージ + 入力フィールド有効化
- **最適化失敗**: エラーメッセージ + 条件緩和の提案

---

## 空状態

- **衣装なし**: イラスト + 「最初の衣装を追加しましょう」+ 大きな追加ボタン
- **イベントなし**: イラスト + 「イベントを作成または参加しましょう」+ 2つのボタン
- **使用履歴なし**: 「まだ使用していません」テキストのみ

---

## ダークモード

- 全画面でダークモード対応
- カラーパレットは自動切り替え
- 画像は透明度を下げて目に優しく (opacity: 0.9)

---

このデザイン仕様に基づき、視覚的に美しく、使いやすいモバイルアプリを実装します。
